# Auth System

Backend-сервис аутентификации и ролевого доступа на Django REST Framework.

## Реализовано

- создан Django-проект и приложения `users`, `access_control` и `business`;
- подключён Django REST Framework;
- настроена загрузка переменных окружения из `.env`;
- добавлен шаблон переменных окружения `.env.example`;
- настроено подключение к PostgreSQL через `psycopg`;
- подключены `pytest` и `pytest-django`;
- создана модель `Role` с уникальным системным кодом;
- модель `Role` зарегистрирована в Django Admin;
- создана таблица `roles`;
- приложение контроля доступа переименовано в `access_control`;
- история миграций и объекты PostgreSQL приведены к имени `access_control`;
- реализована собственная модель пользователя на базе `AbstractBaseUser`;
- модель пользователя подключена через `AUTH_USER_MODEL`;
- email используется как уникальный идентификатор пользователя;
- реализован менеджер создания обычных пользователей и суперпользователей;
- реализована нормализация email;
- пароль хешируется средствами Django перед сохранением;
- пользователь связан с ролью через защищённый `ForeignKey`;
- добавлена initial migration приложения `users`;
- в PostgreSQL создана таблица `users_user` со ссылкой на `roles`;
- data migration создаёт базовые роли `user` и `admin`;
- реализована регистрация через `POST /api/auth/register/`;
- регистрация валидирует обязательные поля, email, совпадение и надёжность пароля;
- email при регистрации нормализуется, а роль `user` назначается автоматически;
- пароли сохраняются через `BCryptSHA256PasswordHasher`;
- создана модель JWT-сессии `AuthSession` с идентификатором `jti`, сроком действия и признаком отзыва;
- настроены JWT-параметры через переменные окружения: секрет, алгоритм и TTL access token;
- реализован вход через `POST /api/auth/login/`: выдаётся access token и создаётся `AuthSession`;
- реализована JWT-аутентификация с проверкой подписи, срока токена, активной сессии, её отзыва и состояния пользователя;
- реализован защищённый профиль `GET /api/auth/me/`;
- реализован выход через `POST /api/auth/logout/`: текущая JWT-сессия отзывается, а токен перестаёт действовать;
- регистрация, login, JWT-сессии, аутентификация, профиль и logout покрыты автоматическими тестами.

## Модель роли

| Поле | Назначение |
| --- | --- |
| `id` | Первичный ключ |
| `code` | Уникальный системный код роли |
| `name` | Отображаемое название |
| `created_at` | Дата создания |

## Модель пользователя

| Поле | Назначение |
| --- | --- |
| `id` | Первичный ключ |
| `first_name` | Имя пользователя |
| `last_name` | Фамилия пользователя |
| `middle_name` | Необязательное отчество |
| `email` | Уникальный email и идентификатор для входа |
| `password` | Хеш пароля, управляемый `AbstractBaseUser` |
| `is_active` | Признак активного пользователя |
| `is_staff` | Доступ к административной части |
| `role` | Ссылка на роль пользователя |
| `created_at` | Дата создания |
| `updated_at` | Дата последнего изменения |
| `deleted_at` | Дата мягкого удаления |

Удаление роли запрещено, пока с ней связан хотя бы один пользователь.


## Модель JWT-сессии

| Поле | Назначение |
| --- | --- |
| `id` | Первичный UUID сессии |
| `jti` | Уникальный идентификатор, включаемый в JWT |
| `user` | Пользователь-владелец сессии |
| `created_at` | Дата создания сессии |
| `expires_at` | Время истечения access token |
| `revoked_at` | Время отзыва сессии; `NULL` для действующей сессии |
